# 投资组合管理模块 - 功能场景描述

## 1. 用户角色与配置

### 1.1 用户角色

本系统采用单一用户角色模式，所有用户拥有相同的系统操作权限，无需进行角色分配与权限配置。

| 角色类型 | 描述 | 权限范围 |
|---------|------|---------|
| 默认用户 | 系统唯一用户角色 | 拥有所有组合管理、查看、导出等全部功能权限 |

### 1.2 用户配置属性

```
用户配置:
├── 基本信息
│   ├── 用户ID（系统生成）
│   ├── 用户名
│   └── 创建时间
├── 偏好设置
│   ├── 默认货币（CNY/USD/HKD）
│   ├── 时区
│   └── 通知设置
└── 组合设置
    ├── 最大组合数量（默认：10）
    └── 单组合最大持仓数（默认：50）
```

## 2. 核心用例流程

### 2.1 组合创建流程

#### 场景：创建新投资组合

**参与者：** 用户

**前置条件：**
- 用户已登录系统
- 用户未达到组合数量上限

**主流程：**

```
1. 用户点击导航栏中的"新建组合"按钮
2. 系统显示组合创建表单
3. 用户输入组合名称（必填，最多50字符）
4. 用户输入组合描述（可选，最多500字符）
5. 用户选择创建模式：
   ├── 手动选择 → 进入步骤 6a
   └── 批量导入 → 进入步骤 6b

6a. 手动选择流程：
    ├── 用户通过代码/名称搜索股票
    ├── 系统显示匹配的股票列表
    ├── 用户选择要添加的股票
    ├── 用户设置每只股票的仓位占比
    │   ├── 选项："平均分配"按钮（自动计算 100%/n）
    │   └── 选项：手动输入（验证：总和 = 100%）
    └── 用户确认持仓

6b. 批量导入流程：
    ├── 用户上传CSV文件
    │   └── 格式：code,entry_price,weight（可选）
    ├── 系统验证文件格式
    ├── 系统显示预览及验证结果
    │   ├── 有效条目：绿色勾选标记
    │   ├── 无效代码：红色叉号及错误信息
    │   └── 缺失价格：黄色警告
    ├── 用户审核并调整
    └── 用户确认导入

7. 用户设置初始资金（可选，用于盈亏计算）
8. 用户点击"创建组合"
9. 系统验证：
   ├── 组合名称唯一性
   ├── 至少包含一个持仓
   └── 仓位占比总和 = 100%
10. 系统创建组合，状态为"活跃"
11. 系统显示成功消息并跳转至组合详情页
```

**后置条件：**
- 数据库中创建新的组合记录
- 创建持仓记录，包含建仓价格和仓位占比
- 保存初始快照至历史记录

**异常流程：**

| 异常情况 | 触发条件 | 处理方式 |
|---------|---------|---------|
| 名称重复 | 组合名称已存在 | 显示错误，建议使用唯一名称 |
| 股票代码无效 | 代码在系统中未找到 | 高亮显示无效行，允许跳过或更正 |
| 仓位占比不匹配 | 总和 ≠ 100% | 显示警告，保存前必须更正 |
| 文件解析错误 | CSV格式无效 | 显示具体错误，提供模板下载 |
| API限流 | 股票数据获取失败 | 加入重试队列，显示待处理状态 |

### 2.2 组合管理流程

#### 场景：查看组合仪表盘

**参与者：** 用户

**主流程：**

```
1. 用户导航至组合管理模块
2. 系统显示组合列表及摘要卡片：
   ├── 组合名称
   ├── 持仓数量
   ├── 当前总市值
   ├── 总盈亏（金额和百分比）
   ├── 最后更新时间
   └── 状态（活跃/已删除）
3. 用户点击组合卡片
4. 系统显示组合详情视图：
   ├── 标题栏：名称、描述、创建日期
   ├── 摘要指标：
   │   ├── 初始总资金
   │   ├── 当前总市值
   │   ├── 总盈亏（金额和百分比）
   │   └── 表现最佳/最差持仓
   ├── 持仓表格：
   │   ├── 代码
   │   ├── 名称
   │   ├── 建仓价格
   │   ├── 当前价格
   │   ├── 仓位占比（%）
   │   ├── 盈亏（%）
   │   └── 状态
   └── 操作菜单
```

#### 场景：编辑组合持仓

**参与者：** 用户

**主流程：**

```
1. 用户在组合详情页点击"编辑"按钮
2. 系统进入编辑模式：
   ├── 可编辑字段高亮显示
   └── 显示操作按钮
3. 用户可执行以下操作：
   ├── 新增持仓
   │   ├── 搜索并选择股票
   │   ├── 设置建仓价格和仓位占比
   │   └── 系统重新计算仓位占比（可选择重新分配）
   ├── 移除持仓
   │   ├── 点击删除图标
   │   ├── 系统标记为"待删除"
   │   └── 确认删除
   ├── 修改持仓
   │   ├── 更新建仓价格
   │   ├── 更新仓位占比
   │   └── 系统验证占比总和
   └── 重新平衡仓位
       ├── 点击"平均分配"
       └── 系统自动调整所有仓位占比
4. 用户点击"保存更改"
5. 系统验证并保存
6. 系统记录变更历史
```

### 2.3 组合删除流程

#### 场景：软删除组合

**参与者：** 用户

**主流程：**

```
1. 用户在组合卡片上点击"删除"按钮
2. 系统显示确认对话框：
   ├── 警告："此操作将隐藏该组合，但保留历史数据"
   ├── 选项："取消" / "删除"
   └── 复选框："同时删除所有历史记录"（默认未选中）
3. 用户确认删除
4. 系统执行：
   ├── 设置 is_deleted = true
   ├── 设置 deleted_at = 当前时间戳
   ├── 保留所有历史记录（除非选中复选框）
   └── 从活跃组合列表中移除
5. 系统显示成功提示
```

#### 场景：恢复已删除组合

**参与者：** 用户

**主流程：**

```
1. 用户在组合列表中点击"显示已删除"开关
2. 系统显示已删除的组合（灰色显示）
3. 用户在已删除组合上点击"恢复"按钮
4. 系统执行：
   ├── 设置 is_deleted = false
   ├── 清空 deleted_at
   └── 恢复至活跃列表
5. 系统显示成功消息
```

### 2.5 每周计算流程

#### 场景：自动每周计算

**参与者：** 系统（定时任务）

**触发条件：** 每周五 18:00（可配置）

**主流程：**

```
1. 调度器触发每周计算任务
2. 系统获取所有活跃组合
3. 对每个组合执行：
   ├── 获取所有持仓的当前价格
   ├── 计算每个持仓的当前盈亏：
   │   ├── 盈亏% = (当前价格 - 建仓价格) / 建仓价格 * 100
   │   └── 加权盈亏 = 盈亏% * 仓位占比
   ├── 计算组合总盈亏：
   │   └── 总盈亏 = Σ(加权盈亏)
   ├── 创建历史快照：
   │   ├── 组合级别记录
   │   └── 各持仓独立记录
   └── 保存至 portfolio_history 表
4. 系统记录完成状态
5. 系统发送通知（如已配置）
```

**异常处理：**

| 异常情况 | 处理方式 |
|---------|---------|
| 价格获取失败 | 重试3次，然后标记为"数据过期" |
| 无活跃组合 | 跳过，记录日志 |
| 数据库错误 | 回滚，通知管理员 |

## 3. 错误处理场景

### 3.1 数据验证错误

| 错误代码 | 场景 | 用户提示信息 | 处理方式 |
|---------|------|-------------|---------|
| PORTFOLIO_001 | 组合名称为空 | "请输入组合名称" | 聚焦到名称输入框 |
| PORTFOLIO_002 | 名称过长 | "名称不能超过50个字符" | 截断或拒绝 |
| PORTFOLIO_003 | 名称重复 | "已存在同名组合，请使用其他名称" | 建议唯一名称 |
| PORTFOLIO_004 | 无持仓 | "组合至少需要包含一个持仓" | 阻止保存 |
| PORTFOLIO_005 | 仓位占比错误 | "仓位占比总和必须等于100%" | 显示当前总和 |
| HOLDING_001 | 股票代码无效 | "未找到股票代码 '{code}'" | 允许跳过或更正 |
| HOLDING_002 | 建仓价格为负 | "建仓价格必须为正数" | 阻止保存 |
| HOLDING_003 | 仓位占比无效 | "仓位占比必须在0到100之间" | 阻止保存 |

### 3.2 系统错误

| 错误代码 | 场景 | 用户提示信息 | 恢复操作 |
|---------|------|-------------|---------|
| SYSTEM_001 | 数据库连接失败 | "保存失败，请重试" | 重试按钮 |
| SYSTEM_002 | 价格接口超时 | "无法获取当前价格" | 使用缓存价格 |
| SYSTEM_003 | 文件上传失败 | "文件上传失败，请检查格式" | 显示模板 |
| SYSTEM_004 | 导出生成失败 | "导出失败，请重试" | 重试按钮 |

### 3.3 业务规则违反

| 规则 | 违反情况 | 系统响应 |
|-----|---------|---------|
| 单用户最大组合数 | 用户尝试创建第11个组合 | 显示限制提示，建议删除已有组合 |
| 单组合最大持仓数 | 用户尝试添加第51个持仓 | 显示限制提示 |
| 计算进行中禁止删除 | 用户在计算过程中尝试删除 | 显示"请稍候"提示 |

## 7. 系统集成点

### 7.1 数据提供者集成

```
组合服务
├── 股票价格获取器
│   ├── AkShare（A股）
```

```
每周计算任务
├── 触发条件：Cron表达式（默认：0 18 * * 5）

│   ├── 数据库连接
│   └── 价格接口可用性
└── 错误处理：
    ├── 重试逻辑（3次尝试）
    ├── 死信队列
    └── 管理员通知
```

## 附录A：术语表

| 术语 | 定义 |
|-----|------|
| 组合 | 作为一组追踪的投资持仓集合 |
| 持仓 | 组合中的单只股票头寸 |
| 建仓价格 | 股票加入组合时的价格 |
| 仓位占比 | 持仓在组合中的百分比配置 |
| 盈亏 | 盈利与亏损，以建仓价格为基准计算的百分比变化 |
| 软删除 | 将记录标记为已删除但不从数据库中移除 |
| 快照 | 特定时间点的组合状态和收益记录 |
